## 檔案格式與命名
- **檔案格式**：RTF 格式 (.rtf)
- **命名規則**：`MADZINE [模組名稱] Manual.rtf`
- **語言版本**：英文、日文、繁體中文

## 標準結構範本

### 1. 標題區塊
```
**MADZINE [模組名稱] Module Manual**
[語言標示 - English/日本語]
```

### 2. Overview 概述區塊
- **目的**：一段話說明模組的核心功能和特色
- **風格**：簡潔明瞭，突出主要特性
- **範例**：
  ```
  **Overview** The [模組名稱] module is a [主要功能描述] with [特殊功能]. 
  It combines [功能A] with [功能B], featuring [特色1], [特色2], and [特色3].
  ```

### 3. Features 功能特色
- 使用條列式說明
- 每項功能以 `-` 開頭
- 重要參數用粗體標示
- 包含數值範圍和預設值
- 範例格式：
  ```
  - **功能名稱** with 詳細說明
  - **參數控制** (範圍, default: 預設值)
  - **連接功能** for 用途說明
  ```

### 4. Controls 控制說明
按照面板佈局分區塊說明：

#### 主要分區
- **Global Section** / **Main Section**
- **Track [編號] Section** (多軌道模組)
- **Modulation Section**
- **Input/Output Section**

#### 控制項格式
```
- **控制項名稱**: 功能說明 (數值範圍, default: 預設值)
- **按鈕名稱**: 功能描述with LED 狀態說明
  - 顏色1: 功能狀態1
  - 顏色2: 功能狀態2
```

### 5. Processing Modes / 工作模式 (選用)
針對有多種工作模式的模組：
- 詳細說明每種模式的運作方式
- 包含步驟化的處理流程
- 說明模式間的差異

### 6. 技術規格 (Technical Specifications)
```
**技術規格**
- 輸入範圍: ±10V (信號類型)
- 輸出範圍: 0-10V (信號類型)
- 參數範圍: 具體數值
- 處理: 32位浮點運算
- 取樣: 相關資訊
```

### 7. 版本資訊 (頁尾)
```
**Version [版本號] MADZINE © 2025**
```

## 寫作風格指導

### 語氣規範
- 使用冷靜、專業的語氣
- 避免使用「完美」、「成功」等詞彙
- 不使用驚嘆號
- 保持客觀的技術描述

### 參數標示規範
- **重要控制項**：使用粗體
- 數值範圍：使用括號標示 `(0.0-1.0, default: 0.5)`
- CV輸入：明確標示電壓範圍和功能
- LED狀態：詳細說明顏色對應的功能

### 分區組織原則
1. **由上到下**：按照面板實際佈局順序
2. **功能群組**：相關控制項歸納在同一區塊
3. **輸入輸出**：分別說明，包含連接線顏色慣例
4. **特殊功能**：獨立說明複雜的處理邏輯

## 模板使用說明

1. **複製基本結構**：使用上述範本作為起點
2. **填入模組資訊**：根據實際模組功能調整內容
3. **調整分區**：根據模組複雜度增減區塊
4. **驗證資訊**：確保所有參數和功能描述正確
5. **格式檢查**：統一字體、對齊和標示方式
